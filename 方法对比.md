# 方法对比

## 文档目的

本文件用于从实验视角整理本仓库涉及的主要强化学习方法，重点回答三个问题：

- 为什么同一个仓库里会同时保留 `Value Iteration`、`Q-Learning`、`SARSA`、`DQN`、`PPO`、`DDPG`、`SAC`
- 不同环境为什么要选不同方法，而不是机械统一成一套算法
- 读者在演示或阅读时，应该优先比较哪些实验，才能最快理解方法差异

这份文档不是算法教材，也不替代各实验 notebook 中的参数说明。它更像一份总览索引，帮助读者先建立方法地图，再进入具体实验。

## 一句话总览

如果只看主线，可以把本仓库的方法分成四层：

1. 规划方法：`Value Iteration`
2. 小规模离散强化学习：`Q-Learning`、`SARSA`、`Monte Carlo`
3. 离散动作深度强化学习：`DQN`、`Double DQN`
4. 现代策略优化与连续控制：`PPO`、`DDPG`、`SAC`

这四层不是互相替代关系，而是问题规模、状态表示方式和工程路线逐步升级的关系。

## 方法分层

### 1. 规划方法

代表方法：

- `Value Iteration`

核心特征：

- 需要已知环境转移结构
- 不靠反复采样试错，而是直接做动态规划
- 更适合小型离散环境

适合解决的问题：

- 状态和动作空间都很小
- 环境模型可以直接访问
- 目标是讲清楚“状态价值”和“最优策略”是什么

本仓库对应实验：

- `02-frozenlake-value-iteration`

它的价值不在于工程扩展性，而在于帮助读者建立价值函数的最基础直觉。

### 2. 表格型强化学习

代表方法：

- `Q-Learning`
- `SARSA`
- `Monte Carlo Control`

核心特征：

- 不需要已知环境模型
- 用表格直接存状态或状态-动作价值
- 非常适合离散、小规模、教学型环境

适合解决的问题：

- 状态空间可枚举
- 动作空间离散
- 目标是解释探索、回报、时序差分更新和策略改进

本仓库对应实验：

- `01-frozenlake-tabular-q-learning`
- `03-cliffwalking-sarsa`
- `04-cliffwalking-q-learning`
- `05-taxi-tabular-q-learning`
- `06-blackjack-monte-carlo-control`

这一层是整个仓库的概念地基。后面的深度强化学习方法，本质上都是在想办法把这里的价值估计或策略优化扩展到更复杂的状态表示上。

### 3. 离散动作深度强化学习

代表方法：

- `DQN`
- `Double DQN`

核心特征：

- 用神经网络近似 Q 函数
- 用经验回放和目标网络提高训练稳定性
- 适合连续状态、离散动作的环境

适合解决的问题：

- 状态空间已经大到不能再用表格表示
- 动作仍然是离散的
- 希望展示从表格方法向深度方法的自然升级

本仓库对应实验：

- `07-cartpole-dqn`
- `08-cartpole-double-dqn`
- `09-acrobot-dqn`
- `10-mountaincar-dqn`
- `15-lunarlander-dqn`

这一层的重点不是“最现代”，而是“方法演进”。它让读者看到：当状态空间不再适合表格时，经典价值学习如何迁移到神经网络框架中。

### 4. 现代策略优化与连续控制

代表方法：

- `PPO`
- `DDPG`
- `SAC`

核心特征：

- 不再局限于表格价值更新
- 更适合复杂控制任务
- 更强调批量采样、设备利用和训练稳定性

适合解决的问题：

- 连续动作控制
- 更高维、更复杂的控制环境
- 需要更强的工程展示效果和更现代的算法路线

本仓库对应实验：

- `11-pendulum-ddpg`
- `12-pendulum-sac`
- `13-mountaincarcontinuous-ddpg`
- `14-mountaincarcontinuous-sac`
- `16-lunarlander-ppo`
- `17-lunarlandercontinuous-sac`
- `18-bipedalwalker-ppo`

这一层承担的是仓库里的“现代主线展示”角色。

## 核心方法差异

### `Value Iteration` vs `Q-Learning`

前者是规划，后者是学习。

- `Value Iteration` 假设环境模型已知，直接迭代最优价值
- `Q-Learning` 不知道环境模型，只能靠和环境交互逐步学习

如果要讲“为什么强化学习需要试错”，看 `Q-Learning`；如果要讲“价值函数到底是什么”，看 `Value Iteration`。

### `Q-Learning` vs `SARSA`

这是本仓库里最值得演示的一组方法对比之一。

- `Q-Learning` 是 off-policy，更新时更偏向下一状态的最优动作
- `SARSA` 是 on-policy，更新时跟随当前实际执行策略

放到 `CliffWalking` 里，这个差异会变得非常直观：

- `Q-Learning` 往往更容易学出贴近悬崖的最短路径
- `SARSA` 往往更容易学出稍微绕远但更稳妥的路径

如果要讲“风险偏好如何体现在策略形状上”，优先演示：

- `03-cliffwalking-sarsa`
- `04-cliffwalking-q-learning`

### `Monte Carlo` vs TD 方法

`Monte Carlo` 和 `Q-Learning` / `SARSA` 的主要区别，在于它什么时候更新价值。

- `Monte Carlo` 要等整局结束后再回头更新
- TD 方法每走一步就能用 bootstrapping 做更新

因此：

- `Monte Carlo` 更直观
- TD 方法通常更高效

如果要讲“完整回合回报”与“逐步时序差分更新”的区别，`Blackjack` 是合适入口。

### `DQN` vs `Double DQN`

这组对比的重点不是方法范式变化，而是经典价值学习里的偏差修正。

- `DQN` 是离散动作深度强化学习的标准起点
- `Double DQN` 通过拆分动作选择和动作评估，缓解 Q 值高估

如果要讲“经典方法如何做工程改进”，优先演示：

- `07-cartpole-dqn`
- `08-cartpole-double-dqn`

### `DQN` vs `PPO`

这组对比代表两条很不同的工程路线。

- `DQN` 是 value-based，适合离散动作
- `PPO` 是 policy optimization，更通用，也更适合并行采样

在本仓库中：

- `DQN` 更多承担经典深度强化学习教学角色
- `PPO` 更多承担现代主线展示角色

如果要讲“为什么后期主线会从 DQN 转向 PPO”，可以直接对照：

- `15-lunarlander-dqn`
- `16-lunarlander-ppo`

### `DDPG` vs `SAC`

这组对比主要发生在连续动作控制里。

- `DDPG` 是经典 deterministic actor-critic
- `SAC` 是更现代的 stochastic actor-critic，并引入熵正则

一般来说：

- `DDPG` 更适合讲连续控制方法怎么从 DQN 思想演化过来
- `SAC` 更适合展示现代连续控制中的稳定性和实用性

如果要讲“经典连续控制方法”和“现代连续控制方法”的差异，优先演示：

- `11-pendulum-ddpg` vs `12-pendulum-sac`
- `13-mountaincarcontinuous-ddpg` vs `14-mountaincarcontinuous-sac`

## 按环境类型选方法

### 小型离散环境

推荐优先看：

- `Value Iteration`
- `Q-Learning`
- `SARSA`
- `Monte Carlo`

原因：

- 状态空间小，表格方法和规划方法都能直接解释清楚
- 更适合做概念教学和风险偏好对比

对应环境：

- `FrozenLake`
- `CliffWalking`
- `Taxi`
- `Blackjack`

### 连续状态、离散动作环境

推荐优先看：

- `DQN`
- `Double DQN`
- `PPO`

原因：

- 状态空间变大后，表格方法不再可行
- 但动作仍然是离散的，因此 value-based 和 policy-based 方法都能进入

对应环境：

- `CartPole`
- `Acrobot`
- `MountainCar`
- `LunarLander`

### 连续动作环境

推荐优先看：

- `PPO`
- `SAC`
- `DDPG`

原因：

- 连续动作无法直接用普通 DQN 处理
- 需要 actor-critic 或策略优化路线

对应环境：

- `Pendulum`
- `MountainCarContinuous`
- `LunarLanderContinuous`
- `BipedalWalker`

## 本仓库最值得讲的几组对照

如果是演示用途，不需要一口气把所有实验都讲完。最有代表性的对照其实就几组。

### 1. `FrozenLake`：规划 vs 学习

对照实验：

- `01-frozenlake-tabular-q-learning`
- `02-frozenlake-value-iteration`

看点：

- 同一个环境，为什么既能用规划方法，也能用强化学习方法
- 已知模型和未知模型下，求解路径有什么本质差异

### 2. `CliffWalking`：保守策略 vs 激进策略

对照实验：

- `03-cliffwalking-sarsa`
- `04-cliffwalking-q-learning`

看点：

- on-policy 和 off-policy 的差异不是抽象定义，而是会直接体现在最终路径形状上
- `SARSA` 更稳，`Q-Learning` 更短更激进

### 3. `CartPole`：经典价值学习改进

对照实验：

- `07-cartpole-dqn`
- `08-cartpole-double-dqn`

看点：

- 为什么同样是 DQN 路线，还需要进一步缓解过估计问题

### 4. `LunarLander`：经典深度方法 vs 现代主线方法

对照实验：

- `15-lunarlander-dqn`
- `16-lunarlander-ppo`

看点：

- `DQN` 和 `PPO` 在中等复杂离散控制任务上的路线差异
- 为什么 `PPO` 更适合承担仓库中的主展示算法角色

### 5. 连续控制：经典 actor-critic vs 现代 actor-critic

对照实验：

- `11-pendulum-ddpg`
- `12-pendulum-sac`
- `13-mountaincarcontinuous-ddpg`
- `14-mountaincarcontinuous-sac`

看点：

- 连续动作场景里，为什么需要从 deterministic policy 走向 stochastic policy
- 为什么现代连续控制更常把 `SAC` 作为主力方法之一

3. `03-cliffwalking-sarsa`

## 演示建议

如果是面试、汇报或作品集展示，不建议平均分配时间。更好的讲法是围绕“方法为什么变化”来讲。

推荐的演示主线：

1. 先用 `FrozenLake` 讲清楚最基础的价值学习
2. 再用 `CliffWalking` 讲清楚 `SARSA` 和 `Q-Learning` 的风险偏好差异
3. 再用 `CartPole` 或 `LunarLander` 说明为什么要从表格方法升级到神经网络方法
4. 最后用 `PPO` 或 `SAC` 说明现代强化学习工程为什么更强调稳定性、并行采样和通用性

如果时间很短，只讲三组就够：

- `01` vs `02`
- `03` vs `04`
- `15` vs `16`

## 总结

本仓库的方法不是随意堆在一起的，而是按问题规模、动作空间和工程路线逐层展开的：

- `Value Iteration` 负责讲清楚规划
- `Q-Learning` / `SARSA` / `Monte Carlo` 负责讲清楚小规模离散强化学习
- `DQN` / `Double DQN` 负责讲清楚深度价值学习
- `PPO` / `DDPG` / `SAC` 负责承担现代控制和连续控制主线

如果只抓一条最核心的理解主线，可以概括成一句话：

先理解价值，再理解策略；先理解小环境，再理解复杂控制；先理解经典方法，再理解现代强化学习工程。
