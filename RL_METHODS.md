# Reinforcement Learning Methods

## 文档目的

本文件用于整理本仓库涉及的主要强化学习方法，并说明它们各自的适用场景、优点、缺点以及在本项目中的推荐使用方式。目标不是罗列术语，而是帮助读者快速理解：为什么不同实验要选不同方法，为什么某些算法更适合做基础教学，某些算法更适合做现代控制实验。

## 方法总览

本仓库当前主要关注以下几类方法：

- 动态规划：Value Iteration
- 表格强化学习：Q-Learning、SARSA
- 回合采样方法：Monte Carlo Control
- 深度价值学习：DQN、Double DQN
- 策略优化：PPO
- 连续控制 actor-critic：DDPG、SAC
- 扩展讨论：GRPO

## 1. Value Iteration

### 方法说明

Value Iteration 是经典动态规划方法。它假设环境的状态转移和奖励结构已知，然后通过反复更新每个状态的最优价值，逐步逼近最优策略。

### 优点

- 思路直接，适合讲清楚“价值函数”到底是什么
- 在小规模离散环境中结果清晰，容易验证正确性
- 可以作为强化学习方法的参照基线

### 缺点

- 需要已知环境模型
- 状态空间一大就不现实
- 不能直接应对连续状态和连续动作问题

### 适用场景

- 小型离散环境
- 教学型实验
- 需要与无模型方法做对照的实验

### 本仓库建议

适合用于 `FrozenLake` 这类环境，主要价值在于建立概念基础，而不是承担项目主展示任务。

## 2. Q-Learning

### 方法说明

Q-Learning 是最经典的无模型强化学习方法之一。它通过不断与环境交互，学习状态-动作价值函数 `Q(s, a)`，并用它指导后续决策。

### 优点

- 概念清晰，容易解释
- 不需要已知环境转移模型
- 很适合作为强化学习入门方法
- 在小型离散问题上实现简单、结果直观

### 缺点

- 原始表格版只能处理离散且较小的状态空间
- 探索策略和收敛速度比较依赖超参数
- 不适合连续动作问题
- 一旦环境变复杂，表格方法很快失去可扩展性

### 适用场景

- `FrozenLake`
- `Taxi`
- 小规模离散控制问题

### 本仓库建议

适合作为基础实验方法保留，用于说明强化学习最原始、最核心的价值更新思想。

## 3. SARSA

### 方法说明

SARSA 也是表格强化学习中的经典方法。它与 Q-Learning 的重要区别在于，SARSA 学的是当前实际策略下的价值，因此属于 on-policy 方法。

### 优点

- 能很好地展示 on-policy 学习思路
- 在一些带风险的环境中，行为通常更保守、更稳健
- 非常适合做与 Q-Learning 的对比实验

### 缺点

- 同样受限于表格表示
- 训练效率和最终性能往往不适合更复杂问题
- 在大规模问题上不具备现实扩展性

### 适用场景

- `CliffWalking`
- 需要比较保守策略与激进策略差异的环境

### 本仓库建议

适合作为理论差异展示实验，尤其适合和 Q-Learning 配套讲解。

## 4. Monte Carlo Control

### 方法说明

Monte Carlo Control 通过完整回合结束后的采样结果来估计状态或动作的价值，而不是像 TD 方法那样一步一步做 bootstrapping 更新。

### 优点

- 逻辑直观，容易理解“用整局结果反推策略”
- 适合回合式任务
- 不需要显式环境模型

### 缺点

- 必须等一个 episode 完整结束才能更新
- 样本方差往往比较大
- 对长回合或复杂问题不够高效

### 适用场景

- `Blackjack`
- 具有天然回合边界的任务

### 本仓库建议

适合作为补充方法保留，用于展示强化学习中不同的价值估计方式。

## 5. DQN

### 方法说明

DQN 用神经网络近似 Q 函数，把表格 Q-Learning 推广到连续状态空间。它通常会配合经验回放和目标网络，以减轻训练不稳定问题。

### 优点

- 是深度强化学习最经典的起点之一
- 非常适合讲解“表格方法如何升级成神经网络方法”
- 在离散动作环境中具有代表性

### 缺点

- 只适合离散动作
- 训练稳定性一般，比较吃实现细节和超参数
- 并行采样和设备利用率通常不如 PPO 这类方法直观
- 在很多控制任务中，作品集展示效果不如更现代的方法

### 适用场景

- `CartPole`
- `Acrobot`
- `MountainCar`

### 本仓库建议

建议保留少量 DQN 实验，用于体现经典深度强化学习路线，但不建议把它作为整个仓库的主线方法。

## 6. Double DQN

### 方法说明

Double DQN 是 DQN 的改进版本，核心目的是缓解 Q 值高估问题。它通过分离动作选择和动作评估的角色，减少过于乐观的价值估计。

### 优点

- 通常比原始 DQN 更稳一些
- 适合作为“经典方法改进版”展示
- 能体现强化学习中估计偏差修正的工程思路

### 缺点

- 本质上仍然是 DQN 路线，限制没有根本改变
- 仍然只适用于离散动作
- 相比 PPO 等方法，项目展示时不够通用

### 适用场景

- `CartPole`
- 其他离散动作控制任务

### 本仓库建议

适合作为对照实验，而不是主方法。

## 7. PPO

### 方法说明

PPO 是现代强化学习中最常用的策略优化方法之一。它通过限制每次策略更新幅度，尽量在“能够持续改进策略”和“避免训练剧烈震荡”之间取得平衡。

### 优点

- 通用性强，离散动作和连续动作都能做
- 工程生态成熟，实现资料多
- 训练通常比早期方法更稳定
- 很适合配合并行环境进行批量采样
- 更容易把 `CUDA` 或 `MPS` 利用起来
- 对作品集展示非常友好，容易讲清楚

### 缺点

- 样本效率通常不如一些强 off-policy 方法
- 超参数仍然需要调节
- 如果环境很简单，可能显得有些“杀鸡用牛刀”

### 适用场景

- `CartPole`
- `LunarLander`
- `BipedalWalker`
- 其他希望做现代主线展示的控制任务

### 本仓库建议

PPO 非常适合作为本仓库的主展示算法。它兼顾稳定性、并行友好性、可解释性和工程可实现性，特别适合 notebook 形式的实验仓库。

## 8. DDPG

### 方法说明

DDPG 是一种面向连续动作空间的 deterministic actor-critic 方法。它可以看作把 DQN 的一些思想延伸到连续控制中，通过 actor 输出连续动作，通过 critic 评估动作价值。

### 优点

- 能处理连续动作问题
- 在一些经典控制任务上有代表性
- 是理解连续控制 actor-critic 体系的好入口

### 缺点

- 对超参数、探索噪声和训练细节较敏感
- 稳定性通常不如 PPO 或 SAC
- 项目展示时，往往不如更现代方法有说服力

### 适用场景

- `Pendulum`
- `MountainCarContinuous`

### 本仓库建议

可以作为连续控制的经典基线保留一两个实验，但不建议作为主线方法。

## 9. SAC

### 方法说明

SAC 全称是 Soft Actor-Critic，是一种现代连续控制算法。它在最大化回报的同时引入熵正则项，鼓励策略在学习早期保持一定探索性，从而提升训练稳定性和策略质量。

### 优点

- 在很多连续控制任务上效果较强
- 通常比 DDPG 更稳
- 样本效率往往不错
- 很适合做现代连续控制代表方法

### 缺点

- 动作是连续 `float` 输出，实现和解释都更重
- 通常需要 actor 和多个 critic，训练计算开销较大
- 虽然样本效率不错，但墙钟时间不一定占优
- 在作品集展示上，有时不如 PPO 那么直观易讲

### 适用场景

- `Pendulum`
- `LunarLanderContinuous`
- 其他连续动作控制任务

### 本仓库建议

适合保留少量代表性实验，用来体现连续控制的现代算法路线，但不建议整个仓库都围绕 SAC 展开。

## 10. GRPO

### 方法说明

GRPO 全称是 Group Relative Policy Optimization。它近年的主要影响力来自大语言模型和多模态模型后训练场景，尤其常被用于带有可验证奖励或相对比较奖励的生成式任务。

### 优点

- 在特定生成式模型训练场景中具有研究和工程价值
- 近年关注度高，具备扩展研究意义
- 适合作为“强化学习方法如何迁移到大模型后训练”的讨论点

### 缺点

- 不是 Gym / Gymnasium 经典控制任务的标准主线方法
- 不能笼统地说它对所有控制任务都更快、更稳定或效果更好
- 直接迁移到传统控制 benchmark 时，方法论和工程路线都不够主流

### 适用场景

- LLM / VLM 后训练
- 奖励可比较、可排序的生成式任务
- 研究型扩展实验

### 本仓库建议

不建议作为本仓库主线算法。若后续确实需要，可以单独作为扩展实验进行讨论，明确说明它是探索性方向，而不是标准控制基线。

## 如何为本仓库选方法

如果目标是做一个结构清晰、对外展示效果好的强化学习实验仓库，建议采用下面的分层思路：

- 基础离散实验：Value Iteration、Q-Learning、SARSA、Monte Carlo
- 经典深度强化学习实验：DQN、Double DQN
- 现代主展示实验：PPO
- 连续控制补充实验：SAC 或少量 DDPG

## 推荐策略

如果要兼顾“好理解、好运行、好展示”，推荐优先级如下：

1. `PPO`
2. `Q-Learning / SARSA`
3. `SAC`
4. `DQN / Double DQN`
5. `DDPG`
6. `GRPO`

这个排序不是理论绝对优劣，而是结合本仓库目标给出的工程型建议。这里的重点不是某个算法在论文里是否更强，而是谁更适合在当前项目里承担“基础教学”“主线展示”或“扩展讨论”的角色。
